HOTEL hotel_id(primary key), hotel_name
TYPE
type_id(primary_key), type_name
SOLUTION (hotel_id(primary_key), hotel_name, #type_id(foreign_key))

The relationship of type to hotel is many to many(1..n). This means many hotels can be of the same type

The relationship of hotel to type table is one to many (1..1) which means many hotels can be of the same type...3 start, 3star hotel, 5 star hotel etc.
type_id in the hotel table is the foreign key from the type table which creates a relationship between both tables
